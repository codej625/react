# Next에서 헷갈리기 쉬운 개념인 서버 액션에 대해

<br />
<br />

* Server Actions?

```
서버에서만 실행되는 함수로, 호출 위치에 따라 동작이 달라진다.

서버 컴포넌트에서 호출 -> 함수 직접 실행
클라이언트 컴포넌트에서 호출 -> Next.js가 HTTP 요청 자동 생성

즉 호출하는 쪽이 어디든 실행은 무조건 서버에서 되는 함수이다.

설명으로는 어려우니 예시를 보자.
```

<br />
<br />
<br />
<br />

1. 서버 컴포넌트에서 호출

```tsx
// app/users/page.tsx
import { createUser } from '@/actions/user';

export default async function Page() {
  await createUser(); // 서버 -> 서버 직접 호출 (HTTP 요청 없음)
  return <div>...</div>;
}
```

```
서버 컴포넌트 -> Server Actions -> DB
(같은 서버 안에서 함수 호출)
```

<br />
<br />
<br />

2. 클라이언트 컴포넌트에서 호출

```tsx
'use client';
import { createUser } from '@/actions/user';

export default function Form() {
  await createUser(); // 클라이언트 -> Next서버로 HTTP 요청 -> 함수 실행
}
```

```
클라이언트 -> HTTP 요청 -> Server Actions -> DB
(fetch와 원리 동일, Next.js가 대신 처리)
```

```
API Route 파일 X
-> Next가 API 서버 역할까지 한다고 가정
```

```
fetch 작성 X 
-> const handleSubmit = async () => { const res = await fetch('/api/user', ... }
```

```
method, headers, body 설정 X (
-> { method: 'POST', headers: { 'Content-Type': 'application/json' }, ...
```

```
res.json() 파싱 X
-> const data = await res.json();
```

```
TypeScript 타입 따로 정의 X
-> 서버 액션 시 사용하는 타입 그대로 사용
```
