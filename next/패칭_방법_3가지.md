# Next.js 패칭 방법 3가지

<br />
<br />

* 상황에 맞는 패칭

---

```
Next는 여러 가지 렌더링과 패칭 방법이 존재한다.

상황에 맞지 않게 사용하면 최적화에서 문제가 되기 때문에
각 패칭 방법마다 사용되는 상황을 알아보자.
```

<br />
<br />
<br />
<br />

1. Server Component (RSC)

<br />

`페이지/컴포넌트 렌더링 시 데이터 필요할 때 (GET)`

```tsx
// app/users/page.tsx
export default async function UsersPage() {
  const res = await fetch(`${process.env.INTERNAL_API_URL}/users`)
  const users = await res.json()

  return <div>{users.map(u => <p key={u.id}>{u.name}</p>)}</div>
}
```

<br />
<br />
<br />

2. Server Action

<br />

`데이터 생성/수정/삭제 (POST/PUT/DELETE)`

```ts
// app/actions/user.action.ts

'use server'

export async function createUser(formData: FormData) {
  const res = await fetch(`${process.env.INTERNAL_API_URL}/users`, {
    method: 'POST',
    body: JSON.stringify({ name: formData.get('name') }),
    headers: { 'Content-Type': 'application/json' }
  })
  return res.json()
}
```

```tsx
// app/users/page.tsx

import { createUser } from '@/app/actions/user.action'

export default function UsersPage() {
  return (
    <form action={createUser}>
      <input name="name" />
      <button type="submit">생성</button>
    </form>
  )
}
```

3. Route Handler + 클라이언트 컴포넌트

<br />

`클라이언트 컴포넌트에서 동적 fetch가 필요할 때`

```ts
// app/api/users/route.ts

export async function GET() {
  const res = await fetch(`${process.env.INTERNAL_API_URL}/users`)
  const data = await res.json()
  return Response.json(data)
}
```

```tsx
// app/users/UserList.tsx

'use client'

export default function UserList() {
  const [users, setUsers] = useState([])

  useEffect(() => {
    fetch('/api/users')  // Next Route Handler 호출
      .then(res => res.json())
      .then(setUsers)
  }, [])

  return <div>{users.map(u => <p key={u.id}>{u.name}</p>)}</div>
}
```

4. 요약

| 용도                  | 패칭 방법          |
|----------------------|------------------|
| 데이터 보여주기          | Server Component |
| 생성/수정/삭제          | Server Action    |
| 클라이언트에서 동적 fetch | Route Handler    |
